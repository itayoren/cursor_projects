Key Objectives

Quantify the Intangible

Translate "seeing what the customer sees" into "time / money saved" via reduced truck rolls and improved First-Time Resolution (FTR).

Native Integration

Leverage Salesforce's Session Tracing Data Model (STDM) so that Sophie's data can be used in Agentforce's Observability features, standard Salesforce Reports, Dashboards, and CRM built-in flows.

Executive Visibility

Provide high-level insights for service leaders while giving granular operational data to team leads.

Assumptions

  • Sophie visual agentic AI is an action within Agentforce's Service agent
  • Sophie's input includes conversation context (customer details, intent summary)
  • Sophie's output data is being stored as part of the Session Tracing Data Model in the customer's Data360

    Output data includes:

    • VisualOutput - {Device: Text, DeviceVersion: Text, Output: Text}
    • ConfidenceScore - Float
    • IsRetryNeeded - Boolean
  • Sophie's data is linked to the Work Order and Service Appointment objects in Service Cloud
  • Visual inputs should be stored for analysis and attached to WOs or Cases if created

Exploration

Affect Agent decision based on Value Meter

Since metrics are being calculated and tracked in near real time, we're able to close the loop back to Agentforce whenever there's a pattern that requires action.

1

Visual Cross-Reference

Cross reference between the visual input and historical data. Standard CRM objects rely on text data or customer descriptions, which can be vague or inaccurate (e.g., a customer saying "the light is off" when it is actually blinking). Not relying solely on customer description, but crossing the visual input reference with historical inputs can enrich the agent and ground it for better decision making.

2

High Confidence Routing

When understanding the user's request at the start of the session, the agent can call an action to query Value Meter's data (i.e. what's the resolution confidence for this device type) and route high confidence resolutions directly to Sophie.

3

Low Accuracy Alert

On a given day, if the accuracy rate is constantly low to more than 70% of the sessions AND 100+ sessions, produce an alert to the agent admin and immediately escalate sessions by skipping Sophie's action triggering.

4

Similar Issues Learning

When Sophie returns a low confidence output on a given device, an invocable action will be triggered to query STDM and look for similar use cases with successful resolution. These can be either Agent or Human resolved solutions and will be proposed to the end user.

Vivint's E2E Use Case

Company Overview

vivint.com

Vivint provides integrated smart home systems (cameras, locks, sensors) that require precise installation and connectivity. Vivint is also an active Agentforce customer.

The Problem

Vivint faces high operational costs due to truck rolls. When a customer says, "My doorbell camera is offline," it is difficult to diagnose over the phone. Often, a technician is dispatched only to find the device was simply unplugged or needed a reset.

The Context

Vivint uses Agentforce to automate support, but they struggle to automate technical hardware troubleshooting because text-based AI cannot "see" the device status LED or wiring.

The Opportunity

By using TechSee, Vivint can virtually visit the home. The challenge is proving the ROI of this technology to their CFO and knowing when it is safe to let an AI Agent handle the ticket autonomously.
